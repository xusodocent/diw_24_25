    Bones pràctiques

    A continuació, es mostren exemples pràctics i principis clau d’accessibilitat web, seguint les directrius de l’Web Content Accessibility Guidelines (WCAG).

    Principis d'Accessibilitat Web

    Perceptible

    • La informació ha de ser fàcil de percebre per a tots els usuaris.
    • Exemple: Proporcionar textos alternatius (alt text) per a les imatges.
    • Evitar colors amb poc contrast que dificulten la lectura.

    Operable

    • Els components del lloc web han de ser fàcils d’utilitzar.
    • Exemple: Assegura que tot el contingut siga navegable amb el teclat.
    • Temps suficient per completar tasques (com formularis sense límits estrictes).

    Comprensible

    • El contingut ha de ser fàcil d’entendre.
    • Exemple: Utilitza un llenguatge senzill i evita tecnicismes innecessaris.
    • Proporciona instruccions clares per a les accions que cal fer, com completar formularis.

    Robusta

    • El lloc web ha de funcionar en diferents navegadors, dispositius i tecnologies assistives.
    • Exemple: Verifica que el lloc siga compatible amb lectors de pantalla com NVDA o JAWS.
    • Utilitza estàndards com HTML5 i etiquetes semàntiques correctes.

    Exemples Pràctics d'Accessibilitat

    1. Text Alternatiu per a Imatges

    • HTML Correcte:
      HTML
      1
      <img src="foto.jpg" alt="Una persona somrient amb un ordinador portàtil">
    • Error Comú:
      HTML
      1
      <img src="foto.jpg" alt="">
    • No proporciona cap informació als usuaris amb discapacitat visual.

    2. Contrast de Color Adequat

    • Bé: Text negre sobre fons blanc (relació de contrast de 7:1).
    • Malament: Text groc clar sobre fons blanc (relació de contrast de 1.5:1).
    • Eina Recomanada: Contrast Checker.

    3. Navegació amb el Teclat

    • Assegura que els usuaris poden moure’s pel lloc utilitzant la tecla TAB i activant elements amb ENTER o ESPAI.
    • Exemple:
      HTML
      1
      <a href="pagina.html" tabindex="0">Enllaç accessible</a>

    4. Àudio i Vídeo amb Subtítols

    • Inclou subtítols en tots els vídeos.
    • Proporciona una transcripció completa del contingut àudio.
    • Exemple d’integració de subtítols:
      HTML
      1
2
3
4
      <video controls>
  <source src="video.mp4" type="video/mp4">
  <track src="subtitols.vtt" kind="subtitles" srclang="ca" label="Valencià">
</video>

    5. Estructura Semàntica

    • Utilitza etiquetes HTML correctes per a millorar l’accés dels lectors de pantalla:
    • Bé:
      HTML
      1
2
3
      <h1>Títol principal</h1>
<h2>Secció secundària</h2>
<p>Açò és un paràgraf.</p>
    • Malament:
      HTML
      1
2
3
      <div class="titol">Títol principal</div>
<div class="seccio">Secció secundària</div>
<p>Açò és un paràgraf.</p>

    6. Atributs ARIA per a Accessibilitat

    • Exemple: Definir un botó personalitzat com a botó accessible.
      HTML
      1
      <div role="button" tabindex="0" aria-pressed="false">Botó personalitzat</div>
    • Atributs comuns:
    • aria-label: Proporciona una etiqueta descriptiva.
    • aria-hidden: Oculta un element del lector de pantalla.

    7. Formularis Accessibles

    • Cada camp de formulari ha d’estar associat amb una etiqueta.
    • Bé:
      HTML
      1
2
      <label for="nom">Nom:</label>
<input type="text" id="nom" name="nom">
    • Malament:
      HTML
      1
      <input type="text" placeholder="Escriu el teu nom">

    Eines per Verificar l’Accessibilitat

    1. Wave: https://wave.webaim.org
    2. Lighthouse: Integrada en el navegador Chrome.
    3. NVDA o JAWS: Per provar la compatibilitat amb lectors de pantalla.
    4. Axe DevTools: Extensió per a navegadors per a auditar accessibilitat.
