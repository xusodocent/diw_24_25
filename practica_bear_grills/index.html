    Pràctica Guiada: Efecte Paral·laxi amb animacions

    Fins ara hem vist com implementar l'efecte Paral·laxi utilitzant capes que es desplacen a diferents velocitats amb l'scroll del navegador. Les animacions ens permeten manipular les propietats dels elements CSS d'una manera dinàmica, i en la present pràctica aconseguirem crear l'efecte Paral·laxi manipulant les diferents capes per a generar un efecte de profunditat gràcies al moviment i la superposició dels diferents elements. El resultat final serà el següent:

    Per a començar la pràctica descarrega el següent fitxer on trobaras les imatges necessaries.

    Genera un document anomenat index.html que allotge l'estructura que es mostra en el següent punt. Adjunta un fitxer anomenat styles.css on definir el codi CSS que s'indica en la pràctica.

    Estructura HTML

    L'estructura que suporta el contingut de la imatge anterior i les animacions és el següent:

    <div class="wrapper">
    <div class="pos text"></div>
    <div class="pos mountain1"></div>
    <div class="pos mountain2"></div>
    <div class="pos mountain3"></div>
</div>

    CSS Inicial

    body

    L'element body és l'element pare de tot el contingut visible en una pàgina web. A continuació es mostra el codi CSS que s'aplicarà sobre l'element i seguidament la seua explicació:

    body {
  /*Dimensions*/
  height: 100%;

  /*Fons*/
  background: url(img/00.jpg);
  background-position: center center;
  background-size: cover;

  /*Altres*/
  margin: 0;
  padding: 0;
}

    Aquest element haurà d'allotjar la imatge que se situa més al fons de la composició (background: url(img/00.jpg)). La particularitat de la capa, i més concretament del seu fons, és que ha d'ocupar tot l'espai disponible (background-size: cover) i ha d'anar centrada tant en l'eix vertical com en l'eix horitzontal (background-size: center center).

    El resultat d'aplicar l'estil serà el següent:

    .wrapper

    Aquesta classe té com a funció principal fer d'embolcall del contingut que forma part de l'animació. A continuació es mostra el codi CSS que s'aplicarà sobre la classe i seguidament la seua explicació:

    .wrapper {

  /*Dimensions*/
  width: 100vw;
  height: 100vh;

  /*Posició*/
  position: relative;

  /*Altres*/
  overflow: hidden;
}
    La posició d'aquest embolcall és relativa (position:relative), amb la qual cosa, en cas d'incloure més elements al disseny, se situarà immediatament a continuació d'aquests, com per exemple en cas d'incloure una capçalera. A més, tot el seu contingut tindrà disponible l'amplada i alçada de l'espai visible del navegador. (width: 100vw i height: 100vh).

    .pos

    Aquesta classe té com a funció principal preparar les capes perquè queden superposades unes a altres. A continuació es mostra el codi CSS que s'aplicarà sobre la classe i seguidament la seua explicació:

    .pos {

  /*Dimensions*/
  width: 100vw;
  height: 100vh;

  /*Posició*/
  position: absolute;
  top: 0;
  left: 0;

}
    La classe s'aplica a les capes amb classe .text, .mountaint1, .mountain2 i .mountain3. Es caracteritza pel seu posicionament absolut (position:absolute), el qual situa totes les capes anteriors, superposades entre elles a partir de la part superior-esquerra del seu contenidor (top: 0 i left: 0). Novament, i respecte a les dimensions, l'amplada i alçada de les capes amb aquesta classe serà l'àrea visible del navegador (width: 100vw i height: 100vh).

    L'ordre de superposició de les capes amb aquesta classe serà l'ordre d'aparició en l'arbre (renderitzat) tal com es mostra a continuació:

    Note

    Si volem alterar l'ordre de superposició podem fer-ho, o bé alterant l'ordre en el codi html, o bé utilitzant la propietat z-index.

    .text

    Aquesta classe conté el text de fons (background: url(img/00.jpg)). A continuació es mostra el codi CSS que s'aplicarà sobre la classe i seguidament la seua explicació:

    .text {
    background: url(img/text.png);
    background-position:top center;
    background-size: 45vw;
    background-repeat: no-repeat;
}
    La imatge del text no ha de cobrir tot l'ample visible del contenidor i s'ha optat per ocupar el 45% d'aquest (background-size: 45vw). Donat que la imatge no ocupa tota l'alçada i amplada és convenient centrar-la horitzontalment i alinear-la verticalment a la part superior (background-position:top center), i a més, cal evitar que aquesta es repetisca en l'espai sobrant (background-repeat: no-repeat). El resultat d'aplicar l'estil a la classe és el següent:

    .mountain1

    Aquesta classe té de fons la imatge la muntanya del fons (background: url(img/01.png)). A continuació es mostra el codi CSS que s'aplicarà sobre la classe i seguidament la seua explicació: 

    .mountain1 {
  background: url(img/01.png);
  background-position: center center;
  background-size: cover;
}
    La imatge ocupa tot l'espai disponible (background: background-size: cover) i està centrada en els dos eixos (background-position: center center). El resultat d'aplicar l'estil a la classe és el següent:

    .mountain2

    Aquesta classe té de fons la imatge la muntanya lateral (background: url(img/02.png)). A continuació es mostra el codi CSS que s'aplicarà sobre la classe i seguidament la seua explicació: 

    .mountain2 {
  background: url(img/02.png);
  background-position: left bottom;
  background-size: 120%;
  background-repeat: no-repeat;
}

    Per les seues característiques, s'ha decidit que la mida de la imatge siga un 20% major que la seua mida original (background: background-size: 120%). Com no ocupa tot l'espai disponible, cal evitar que es repetisca (background-repeat: no-repeat). Per últim, la imatge ha de quedar alineada a la part inferior-esquerra (background-position: left bottom).

    El resultat d'aplicar l'estil a la classe és el següent:

    .mountain3

    Aquesta classe té de fons la imatge del protagonista de la composició (background: url(img/03.png)). La imatge ha d'ocupar tot l'espai disponible (background: background-size: cover) i ha de quedar centrada en els dos eixos (background-position: center center). A continuació es mostra el codi CSS.

    .mountain3 {
  background: url(img/03.png);
  background-position: center center;
  background-size: cover;
}

    Animació dels elements

    Arribats a aquest punt, ja tenim l'estructura i estils necessaris per a començar a animar els difents elements. Fins ací el seu resultat ha de ser el següent:

    Animació move

    Una de les animacions que s'utilitza per a l'efecte Paral·laxi és el moviment de les capes per a fer la sensació de profunditat. Per a això mitjançant la regla @keyframes es crea una animació anomenada move que s'encarregarà d'eliminar qualsevol translació i, al mateix temps, restableix l'opacitat aplicada sobre un element.

    @keyframes move {

  100% {
    transform: translate3d(0, 0, 0);
    opacity: 1;
  }

}

    L'anterior codi elimina qualsevol translació (!#css transform: translate3d(0, 0, 0);) i restableix l'opacitat (opacity: 1), en conseqüència, l'estat inicial dels elements ha de tenir un valor inicial diferent per a aquestes propietats al de l'estat final, en cas cotrari no s'apreciarà cap animació. Per això establirem els següents valors a la següent classe:

    .text {
    transform: translate3d(0, 30px, 0);
    opacity: 0;
    animation: move 2s ease-out;
    animation-fill-mode: forwards;
}

    Del codi anterior s'observa que la classe .text tindrà inicialment un desplaçament de 30px sobre l'eix y situant-se 30px per davall de la seua posició original (transform: translate3d(0, 30px, 0)) i, a més, serà totalment transparent (opacity:0). L'animació move durarà 2 segons (animation: move 2s ease-out). Una vegada finalitze l'animació, les modificacions aplicaciodes sobre l'element hauran de romandre en el temps (animation-fill-mode: forwards).

    El resultat de l'animació move sobre la classe .text és el següent:

    Note

    La propietat animation-fill-mode:forwards manté els valors de l'últim fotograma de l'animació una vegada ha finalitzat.

    Animació scale

    L'altra animació que s'utilitza consisteix en la transformació del canvi d'escala en 3 dimensions (scale3d()). Cada element subjecte a l'animació partirà d'una escala modificada i acabarà en la seua proporció original. A més, com en l'animació anterior, el valor d'opacitat de l'element es restableix al valor per defecte. Tot seguit es mostra el @keyframes de l'animació scale:

    @keyframes scaler {

  100% {
    transform: scale3d(1, 1, 1);
    opacity: 1;
  }

}

    Els elements sobre els quals aplicarem l'animació anterior es mostren a continuació, i com es pot observar, tenen tant l'escala com l'opacitat modificades. Després de l'animació recuperaran els seus valors originals i persistiran. El codi per a cada classe és el següent:

    .mountain1 {
  transform: scale3d(1.1, 1.1, 1.1);
  opacity: 0.2;
  animation: scaler 1s ease-out;
  animation-fill-mode: forwards;
}

.mountain2 {
  transform: scale3d(1.3, 1.3, 1.3);
  opacity: 0.1;
  animation: scaler 1.2s ease-out;
  animation-fill-mode: forwards;
}

.mountain3 {
  transform: scale3d(1.4, 1.4, 1.4);
  opacity: 0.1;
  animation: scaler 1.4s ease-out;
  animation-fill-mode: forwards;
}
